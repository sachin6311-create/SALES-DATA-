# SalesData Database Project

This project creates a `SalesData` table in MySQL and populates it with 10,000 rows of random customer purchase data.

## Table Structure

### SalesData Table

| Column | Type | Description |
|--------|------|-------------|
| CustomerID | INT (Primary Key) | Unique customer identifier (auto-increment) |
| Name | VARCHAR(100) | Customer's full name |
| Age | INT | Customer's age |
| City | VARCHAR(50) | Customer's city |
| PurchaseAmount | DECIMAL(10, 2) | Amount spent in the purchase |
| PurchaseDate | DATE | Date of the purchase |

## Files

### 1. `schema.sql`
Creates the `SalesData` table with proper structure and indexes.

**How to use:**
```bash
mysql -u username -p database_name < schema.sql
```

### 2. `generate_data.py`
Python script that generates 10,000 rows of random sales data and creates a MySQL INSERT statement.

**Requirements:**
- Python 3.6+

**How to use:**
```bash
python3 generate_data.py
```

This will create a file named `insert_10000_rows.sql` containing the INSERT statement.

### 3. `insert_data.sql`
Sample INSERT statement with 50 rows (for testing).

### 4. `insert_10000_rows.sql`
Generated by the Python script - contains INSERT statement for 10,000 rows.

## Setup Instructions

### Step 1: Create the Table
```bash
mysql -u your_username -p your_database_name < schema.sql
```

### Step 2: Generate Random Data
```bash
python3 generate_data.py
```

This creates `insert_10000_rows.sql` with 10,000 random records.

### Step 3: Insert Data
```bash
mysql -u your_username -p your_database_name < insert_10000_rows.sql
```

## Alternative: Single Command

If you want to do everything at once:

```bash
# Generate data
python3 generate_data.py

# Create schema and insert data
mysql -u your_username -p your_database_name < schema.sql
mysql -u your_username -p your_database_name < insert_10000_rows.sql
```

## Data Characteristics

- **Total Records**: 10,000
- **Names**: Random combinations of first and last names
- **Ages**: Random between 18 and 80
- **Cities**: 50 major US cities
- **Purchase Amount**: Random between $50.00 and $1000.00
- **Purchase Date**: Random dates throughout 2024

## Query Examples

### Get total sales by city
```sql
SELECT City, SUM(PurchaseAmount) as TotalSales, COUNT(*) as CustomerCount
FROM SalesData
GROUP BY City
ORDER BY TotalSales DESC;
```

### Find customers by age range
```sql
SELECT * FROM SalesData
WHERE Age >= 25 AND Age <= 35
ORDER BY PurchaseAmount DESC;
```

### Get average purchase amount by city
```sql
SELECT City, AVG(PurchaseAmount) as AvgPurchase
FROM SalesData
GROUP BY City
ORDER BY AvgPurchase DESC;
```

### Get customers with high purchase amounts
```sql
SELECT * FROM SalesData
WHERE PurchaseAmount > 500
ORDER BY PurchaseDate DESC;
```

## Notes

- The `CustomerID` field is auto-increment, so you don't need to specify it in INSERT statements
- Indexes are created on `CustomerID` and `PurchaseDate` for better query performance
- All purchase amounts are stored as DECIMAL for financial accuracy
- Dates are stored in YYYY-MM-DD format
